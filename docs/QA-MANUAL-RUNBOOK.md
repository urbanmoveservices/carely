# Vaidya GPT — Manual QA runbook

Use this guide for checklist items that cannot be verified by API-only automation.

## Before you start

1. Start the app: `npm run dev` (http://localhost:7111)
2. Run automation: `npm run test:all` or `npm run test:qa:auto`
3. Open the checklist: http://localhost:7111/admin/qa-checklist (admin login)
4. Refresh this file: `npm run test:qa:manual-guide` (lists current **pending** items)

## Testing levels

| Command | Purpose |
|---------|---------|
| `npm run test:smoke` | DB, env, manifest, assets |
| `npm run prod:check` | Production readiness |
| `npm run test:e2e` | Upload/OCR, auth, isolation (JPEG fixtures) |
| `npm run test:qa:auto` | Auto-mark checklist from scripts + E2E report |
| `npm run test:all` | All of the above + build |
| `npm run test:qa:manual-guide` | Print pending manual steps |

## Image fixtures (E2E upload)

- `test-fixtures/lab-page-1.jpeg` … `lab-page-4.jpeg`
- Optional: `test-fixtures/sample-report.pdf`, `sample-report.docx` for document upload automation

## Common manual areas

- **Mobile / PWA:** layout, hamburger, install prompt, push notifications
- **Auth email flows:** forgot password, reset, verify email (use dev email logs)
- **Report UI:** charts, PDF header branding, share link expiry
- **Language:** Hindi UI without refresh, Urdu RTL, translation toast behavior
- **Billing:** Razorpay checkout, test payment, plan activation, payment history
